# 销售培训学习系统 PRD

**产品名称**：待定（暂称 SalesLearn)
**版本**：v1.0 MVP
**作者**：Funny Liu
**日期**:** 2026-04-21
**目标用户**：自家汽车美容门店销售团队（MVP 阶段）→ 后续 SaaS 化面向同行

---

## 1. 产品定位

基于**费曼学习法**的 AI 销售培训系统,解决汽车美容行业销售"背得出参数、讲不清价值"的核心痛点。

**核心差异化**:
- 不只考"识别"(选择题),更考"输出"(讲解能力)
- AI 扮演真实客户追问,模拟实战场景
- 三层评估体系,最终和业绩转化挂钩

---

## 2. 核心用户场景

**场景 1 - 新人入职培训**
新销售入职 → 学习图文知识 → 做题过基础关 → 费曼讲解给 AI 听 → 解锁实战模拟 → 主管查看评估报告决定是否上岗

**场景 2 - 老员工能力提升**
针对错题本和讲解薄弱点定向训练,每周生成能力报告

**场景 3 - 新产品/新政策上线**
主管上传新资料 → AI 切分生成图文卡 + 测试题 → 全员限时学习考核

---

## 3. 系统架构(4 模块 + 3 层评估)

```
┌─────────────────────────────────────────────────┐

└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│  模块二:图文学习(员工端)                        │

└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐

└─────────────────────────────────────────────────┘
                      ↓

│  模块四:费曼讲解(输出层,核心差异化)             │
```

## 4. 模块详细设计

### 4.1 模块一:知识库录入与管理

#### 功能清单
- **资料上传**:支持 PDF / Word / TXT / Markdown / 成交录音转录稿
- **AI 自动切分**:调用 LLM 将长文档切分为独立知识点
- **结构化生成**:每个知识点生成
  - 标题
  - 核心要点(3-5 条)
  - 详细说明
  - 真实案例/话术示例
  - **常见误区**(关键,用于模块三生成干扰项)
  - 配图(v1.0 支持手动上传,v2.0 考虑 AI 生图)
  - 分类标签(产品知识/客户异议/成交话术/客户心理)
- **人工审核工作流**
  - AI 生成结果进入"待审核"状态
  - 主管可编辑/通过/打回
  - 通过后正式入库,员工可见

#### 数据模型
```
knowledge_base
  - id
  - title
  - category (enum: product / objection / closing / psychology)
  - key_points (array)
  - content (text)
  - examples (text)
  - common_mistakes (text)  -- 模块三用
  - images (array of urls)
  - status (enum: draft / reviewing / published)
  - created_by, reviewed_by
  - created_at, updated_at
```

---

### 4.2 模块二:图文学习

#### 功能清单
- 按分类浏览知识卡片
- 支持搜索、收藏
- **学习行为记录**:停留时长、滑动深度、完成标记
- 学完一个知识点后,引导进入模块三的对应题目

#### 交互要点
- 卡片左右滑动浏览(类似抖音上下滑,但横向)
- 卡片底部:"去测试一下" 按钮 → 跳转模块三

---

### 4.3 模块三:卡片测试(识别层)

#### AI 出题策略(关键)

**题型分布(销售培训场景)**:

| 题型 | 占比 | 考察目标 |
|---|---|---|
| 记忆题 | 20% | 产品参数、政策条款 |
| 理解题 | 20% | 原理、因果关系 |
| **应用题** | 30% | 场景判断("客户 X 情况,怎么推荐?") |
| **辨析题** | 30% | 易混淆点(竞品对比、相近型号差异) |

**干扰项设计规则**:
- ❌ 禁止:明显离谱的选项(如"保修 100 年")
- ✅ 要求:每个错误选项都来自真实易错点
  - 优先从知识点的 `common_mistakes` 字段抽取
  - 其次使用相邻知识点的参数作为干扰项

#### 出题 Prompt 框架(伪代码)
```
你是汽车美容行业销售培训专家。基于以下知识点,生成 {n} 道{题型}题。

【知识点】
{knowledge_content}

【常见误区】
{common_mistakes}

【出题要求】
1. 题目必须贴近销售实战场景(客户对话、异议处理)
2. 每道题 4 个选项,1 个正确
3. 错误选项必须基于"常见误区"或真实易错点,禁止离谱干扰
4. 每个选项附上"为什么对/错"的解析

输出 JSON 格式:
{
  "questions": [
    {
      "type": "application",
      "question": "...",
      "options": ["A","B","C","D"],
      "correct": "B",
      "explanations": {"A":"...","B":"...","C":"...","D":"..."}
    }
  ]
}
```

#### 题目审核机制
- AI 生成 → 待审核池
- 主管一键:通过 / 修改 / 打回
- **精品题标记**:标为精品的题目作为 AI 出新题的 few-shot 样例

#### 错题本(艾宾浩斯间隔复习)
- 答错 → 自动进入错题本
- 复现间隔:1 天 / 3 天 / 7 天 / 15 天
- 连续答对 3 次 → 移出错题本
- **关键联动**:错题本中的知识点自动解锁对应**费曼讲解任务**

---

### 4.4 模块四:费曼讲解(核心模块)

#### 产品形态
**按住录音 → 松开发送 → 语音转文字 → AI 评分 → 追问对话**

#### 两阶段评估(MVP 阶段两者都做)

**阶段 A:关键点覆盖评分**
- 用户讲解内容 vs 知识库标准要点
- AI 对比覆盖度,输出:
  - ✅ 讲到的关键点
  - ⚠️ 遗漏的关键点
  - ❌ 讲错/讲反的点
- 适合初学者打基础

**阶段 B:AI 客户追问实战**
- AI 扮演 3 种客户人设:
  - **小白型**:问"这是啥意思?""为什么要这个?"
  - **比价型**:问"隔壁店便宜 500,凭啥选你?"
  - **懂车型**:用专业问题挑刺("你说的抗黄变指标是 ΔE 还是 YI?")
- 多轮对话(3-5 轮),员工必须讲到 AI "被说服"
- 通过条件:AI 人设给出"好,我考虑一下" / "有道理" 等信服回应

#### 解锁逻辑
- 基础知识点:直接进入阶段 A
- 通过阶段 A(≥80 分)→ 解锁阶段 B
- 错题本中的知识点:**强制走阶段 B**,让员工真正讲透

#### 评分维度(4 维)

| 维度 | 权重 | 说明 |
|---|---|---|
| 完整度 | 30% | 关键点覆盖率 |
| 准确度 | 30% | 有无事实错误、口误 |
| 清晰度 | 20% | 语言组织、逻辑连贯性 |
| 类比恰当度 | 20% | 有无用通俗类比,是否准确 |

#### 评分 Prompt 框架
```
你是资深汽车美容销售培训师。员工刚才口述讲解了以下知识点,请从 4 个维度打分(0-100)并给出改进建议。

【员工应该讲清楚的内容】
{knowledge_standard}

【员工实际讲解】
{user_transcript}

【评分要求】
1. 完整度:对照关键点列表,打勾讲到的、标红遗漏的
2. 准确度:列出任何事实错误
3. 清晰度:评价语言是否让非专业人听得懂
4. 类比恰当度:有无用类比,类比是否准确

输出 JSON:
{
  "scores": {"complete": 85, "accurate": 90, "clear": 70, "analogy": 60},
  "covered_points": [...],
  "missed_points": [...],
  "errors": [...],
  "suggestions": "..."
}
```

#### 技术栈对接
- **录音**:前端 MediaRecorder API
- **语音转文字**:接入第三方 API(阿里云 / 腾讯云 / Whisper API,你自定)
- **LLM 评分**:Claude Sonnet / GPT-4 / Kimi(复用你已有的多模型方案)

## 5. 评估体系(四层)

| 层级 | 指标来源 | 权重 |
|---|---|---|
| 学习完成度 | 模块二:卡片浏览完成率 | 10% |
| 识别掌握度 | 模块三:测试总分 | 25% |
| **讲解能力** | 模块四:费曼评分(4 维加权) | 45% |
| 业绩转化 | 门店 CRM/POS 成交数据(v2) | 20% |

**报表视图**:
- 员工端:个人能力雷达图、薄弱环节提示
- 主管端:团队排行、薄弱知识点分布、个人成长曲线

## 6. 技术架构

### 6.1 部署方案
- **前端**:Next.js 14 (App Router) + TailwindCSS + shadcn/ui
- **部署**:Vercel
- **数据库**:Neon (Postgres Serverless)
- **ORM**:Drizzle
- **认证**:NextAuth.js(员工账号 + 主管账号两种角色)
- **文件存储**:Vercel Blob 或 S3(知识库图片、录音文件)
- **LLM 调用**:
  - 知识切分 + 出题:Claude Sonnet 4 via API
  - 费曼评分:Claude Sonnet 4
  - 备选方案:Kimi K2 API(成本优化)
- **语音转文字**:阿里云 ASR 或 OpenAI Whisper API

### 6.2 核心表设计(简版)
```
users (id, name, role, shop_id, created_at)
knowledge_base (见 4.1)
questions (id, knowledge_id, type, content, options, correct, status, quality_tag)
user_learning_progress (user_id, knowledge_id, view_duration, completed_at)
user_test_records (user_id, question_id, answer, is_correct, answered_at)
user_feynman_records (user_id, knowledge_id, stage, transcript, audio_url,
                      scores_json, ai_feedback, created_at)
error_book (user_id, knowledge_id, next_review_at, review_count, correct_streak)
```

## 8. SaaS 化预留设计

MVP 阶段虽自用,但架构需预留:
- **多租户隔离**:数据表都带 `shop_id` / `tenant_id`
- **知识库与引擎解耦**:每家店传自己的知识,系统逻辑通用
- **行业知识库模板市场**(第三收入流):未来可售卖通用模板(如"漆面膜话术模板包")
- **定价模型**:平台费 + 费曼讲解次数计费(参考 SalesCoach Pro 教训)

   “用腾讯云”

   “是学习底座”

## 10. 风险与对应

| 风险 | 应对 |
|---|---|
| AI 出题质量不稳 | 强制人工审核 + 精品题 few-shot |
| 费曼评分员工觉得不公 | 评分全透明,展示 AI 判断依据;主管可人工覆盖 |
| 员工抵触"被 AI 考核" | 主管话术:"AI 只是陪练,目的是帮你成单" |
| 语音转文字错字多 | UI 显示转写结果,员工可手动修正后再提交评分 |
| LLM 成本超预算 | 短文本用 Haiku/Kimi,长文本和评分用 Sonnet |
