# SalesLearn - 产品需求文档 (PRD)

## 项目概述

**项目名称**：SalesLearn（销售培训学习系统）
**项目简介**：基于费曼学习法的 AI 销售培训系统，解决汽车美容行业销售"背得出参数、讲不清价值"的核心痛点
**产品定位**：SalesCoach Pro 的学习底座模块
**文档版本**：1.0.0
**最后更新**：2026-04-21

---

## 1. 背景与目标

### 1.1 项目背景
汽车美容行业销售团队普遍存在"背得出参数、讲不清价值"的痛点。传统培训方式以选择题考核为主，只考"识别"不考"输出"，导致销售在实际客户沟通中无法有效传达产品价值。

### 1.2 目标用户
- **MVP 阶段**：自家汽车美容门店销售团队
- **后续阶段**：SaaS 化面向同行门店

### 1.3 核心目标
- 目标1：通过费曼讲解训练，提升销售的产品讲解能力
- 目标2：通过 AI 客户追问模拟，提升实战应对能力
- 目标3：建立三层评估体系，最终与业绩转化挂钩

### 1.4 核心差异化
- 不只考"识别"(选择题)，更考"输出"(讲解能力)
- AI 扮演真实客户追问，模拟实战场景
- 三层评估体系，最终和业绩转化挂钩

---

## 2. 核心用户场景

### 场景 1 - 新人入职培训
新销售入职 → 学习图文知识 → 做题过基础关 → 费曼讲解给 AI 听 → 解锁实战模拟 → 主管查看评估报告决定是否上岗

### 场景 2 - 老员工能力提升
针对错题本和讲解薄弱点定向训练，每周生成能力报告

### 场景 3 - 新产品/新政策上线
主管上传新资料 → AI 切分生成图文卡 + 测试题 → 全员限时学习考核

## 3. 系统架构（4 模块 + 3 层评估）

```
┌─────────────────────────────────────────────────┐
│  模块一：知识库录入与管理（管理端）                │
│  主管上传 → AI 切分 → 人工审核 → 入库            │
└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│  模块二：图文学习（员工端）                        │
│  浏览知识卡片 → 记录学习行为                     │
└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│  模块三：卡片测试（识别层）                        │
│  AI 自动出题 → 作答 → 错题本 → 间隔复习          │
└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│  模块四：费曼讲解（输出层，核心差异化）             │
│  语音讲解 → 转文字 → AI 评分 → 客户追问实战      │
└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│  评估体系：学习完成度 + 测试分 + 讲解力 + 业绩    │
└─────────────────────────────────────────────────┘
```

## 4. 模块详细设计

### 4.1 模块一：知识库录入与管理

**功能清单**：
- 资料上传：支持 PDF / Word / TXT / Markdown / 成交录音转录稿
- AI 自动切分：调用 LLM 将长文档切分为独立知识点
- 结构化生成：标题、核心要点、详细说明、案例话术、常见误区、配图、分类标签
- 人工审核工作流：AI 生成 → 待审核 → 主管编辑/通过/打回 → 正式入库

**分类标签**：产品知识 / 客户异议 / 成交话术 / 客户心理

### 4.2 模块二：图文学习

**功能清单**：
- 按分类浏览知识卡片
- 支持搜索、收藏
- 学习行为记录：停留时长、滑动深度、完成标记
- 学完后引导进入对应题目

**交互要点**：卡片左右滑动浏览，底部"去测试一下"按钮跳转模块三

### 4.3 模块三：卡片测试（识别层）

**题型分布**：
| 题型 | 占比 | 考察目标 |
|---|---|---|
| 记忆题 | 20% | 产品参数、政策条款 |
| 理解题 | 20% | 原理、因果关系 |
| 应用题 | 30% | 场景判断 |
| 辨析题 | 30% | 易混淆点、竞品对比 |

**干扰项设计**：禁止明显离谱选项，错误选项必须来自真实易错点

**错题本（艾宾浩斯间隔复习）**：
- 复现间隔：1天 / 3天 / 7天 / 15天
- 连续答对 3 次移出错题本
- 错题本知识点自动解锁费曼讲解任务

### 4.4 模块四：费曼讲解（核心模块）

**产品形态**：按住录音 → 松开发送 → 语音转文字 → AI 评分 → 追问对话

**阶段 A - 关键点覆盖评分**：讲解内容 vs 知识库标准要点，输出覆盖度评估

**阶段 B - AI 客户追问实战**：
- 3 种客户人设：小白型 / 比价型 / 懂车型
- 多轮对话（3-5 轮），员工必须讲到 AI "被说服"

**解锁逻辑**：基础 → 阶段 A → 通过(≥80分) → 阶段 B；错题本强制走阶段 B

**评分维度**：完整度(30%) + 准确度(30%) + 清晰度(20%) + 类比恰当度(20%)

## 5. 评估体系（四层）

| 层级 | 指标来源 | 权重 |
|---|---|---|
| 学习完成度 | 模块二：卡片浏览完成率 | 10% |
| 识别掌握度 | 模块三：测试总分 | 25% |
| 讲解能力 | 模块四：费曼评分 | 45% |
| 业绩转化 | 门店 CRM/POS 成交数据(v2) | 20% |

**报表视图**：
- 员工端：个人能力雷达图、薄弱环节提示
- 主管端：团队排行、薄弱知识点分布、个人成长曲线

## 8. SaaS 化预留设计

- 多租户隔离：数据表带 `shop_id` / `tenant_id`
- 知识库与引擎解耦：每家店传自己的知识，系统逻辑通用
- 行业知识库模板市场：未来可售卖通用模板
- 定价模型：平台费 + 费曼讲解次数计费

## 10. 风险与应对

| 风险 | 应对 |
|---|---|
| AI 出题质量不稳 | 强制人工审核 + 精品题 few-shot |
| 费曼评分员工觉得不公 | 评分全透明，展示 AI 判断依据；主管可人工覆盖 |
| 员工抵触"被 AI 考核" | 主管话术："AI 只是陪练，目的是帮你成单" |
| 语音转文字错字多 | UI 显示转写结果，员工可手动修正后再提交评分 |
| LLM 成本超预算 | 短文本用 Haiku/Kimi，长文本和评分用 Sonnet |
